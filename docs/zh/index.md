# csgo_gc

csgo_gc 是面向旧版 CS:GO 的本地 Game Coordinator 实现。它会把游戏的 GC 流量重定向到进程内的替代实现，让原本依赖 Valve CS:GO GC 的功能可以在不依赖中心服务器的情况下继续工作。

::: warning 项目状态
csgo_gc 仍未完成，尚不适合普通用户直接稳定使用。实验前请备份游戏文件和库存数据。
:::

## 它恢复了什么

当前项目已经支持一批和 GC 强相关的功能：

- 通过 `csgo_gc/inventory.txt` 编辑本地库存。
- 通过可选的本地 Source RCON 接口实时更新库存。
- 物品装备和负载更新。
- 武器箱、贴纸胶囊、补丁包、涂鸦箱、音乐盒和纪念包开启。
- 纪念品生成、贴纸、补丁、命名牌、涂鸦、音乐盒、武器 StatTrak 和音乐盒 StatTrak MVP 计数传播。
- 仓库单元、StatTrak 交换、汰换合同和游戏内商店购买。
- 可用的大厅、专用服务器支持，以及面向 csgo_gc 服务器的服务器浏览器路径。
- 基于 Steam P2P 的客户端与服务器网络通信。

## 从哪里开始

如果你想作为玩家或服务器管理员测试带有本地 GC 功能的 CS:GO，请从[用户指南](/zh/user/)开始。

如果你想构建项目、研究协议行为或贡献代码，请从[开发者指南](/zh/developer/)开始。

## 它不做什么

csgo_gc 不实现匹配系统。匹配需要中心化服务，这不在项目范围内。
