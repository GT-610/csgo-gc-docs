# 服务器和大厅

CSGO-GC 使用 Steam P2P 接口进行网络通信，并实现了足够的 GC 行为来支持大厅、服务器浏览器可见性、SO cache 转发，以及部分和游戏物品状态相关的行为。

起源引擎游戏的本地游玩本质上是**游戏自动在本地开服务器并连接**，因此其相关行为与连接远程服务器完全一致。本地游玩时加载的过程，和连接远程服务器一样，我们也当作“连接服务器”的过程。

## 匹配

匹配并不是由 GC 完成，而是在独立的中心服务器上，需要独立的协调机制。GC 理论上可以通过重定向匹配服务器流量，并默认返回匹配到指定服务器，但此实现非常复杂，目前不在 GC 的实现计划内。

后续我们可能会考虑如何实现，最终成品可能是先实现一个自由且开源的匹配中心服务器，让社区部署，玩家自行选择接入哪个匹配服务器，从而在哪个服务器上游玩。


## 服务器上的物品状态

ServerGC 路径会验证并转发部分客户端 SO cache 消息。它还支持通过客户端到服务器路径传播 StatTrak 音乐盒 MVP 计数状态。这是本项目的本地行为，不是 Valve 官方后端行为。

如果您在连接服务器的时候查看控制台，您可以看到 `Sending socache` 之类的日志。这就是 GC 在通过 Steam P2P 向服务器发送库存信息。

有时候您可能遇到进入支持 GC 的服务器后仍无法显示库存的问题。绝大部分原因是库存发送失败（无 `Sending socache` 日志输出），或者库存发送过晚（此输出在显示服务器状态日志的输出之后）导致的。此类问题一般是您和目标服务器的网络连接存在障碍。如果您通过 Steam P2P 连接您的好友的服务器（例如接受好友请求，组成房间开始游戏），或者所有玩家的 NAT 类型都较差（NAT4），我们建议您使用虚拟组网工具或 VPN 组网后游玩。
