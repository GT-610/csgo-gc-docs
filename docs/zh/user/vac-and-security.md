# VAC 与服务器安全边界

本页说明的是 CSGO-GC 当前可验证的实际安全边界，不是对 Valve 未来处置方式或第三方服务器策略的保证。我们也**不会对不当使用造成的任何损失负责**。

## 项目会修改什么

客户端启动器会把 CSGO-GC 加载到游戏进程中，并 hook Steam 和 Game Coordinator 接口，以便在本地补回已经缺失的 GC 行为。因此杀毒软件可能将其误报为恶意程序；同时，CSGO-GC 也不能被当作只读数据补丁。

CSGO-GC 服务器不会启用 VAC。GC 服务器路径会清除 Steam 服务器的 `secure` 标志。也就是说，不是所有社区服务器都会禁用 VAC，但是所有 CSGO-GC 服务器都禁用 VAC 。

## 项目不修改什么

从 `steam_hook.cpp` 看，项目没有接管 VAC/Steam 身份认证：

- 客户端连接服务器时，`bSecure` 原样传给 Steam。
- `GetAuthSessionTicket` 仍调用原始 `ISteamUser`。
- `BeginAuthSession` 仍调用原始 Steam 实现。
- 服务端认证也先调用原始 `BeginAuthSession`，仅在认证成功后把客户端交给本地 GC 网络层。

也就是说，只有 GC/库存消息被本项目替换，Steam 身份认证、服务器安全状态，以及 VAC 检测，仍然走官方路径，Steam 看来仍然是一切正常。GC 的替换启动器跳过了部分 `trusted-mode` 启动检查，并向 `LauncherMain` 传入 `true`，但之后服务器认证依然是真实的 Steam secure 会话。

## GC 的 Hook 和外挂 Hook 有什么不同？

CSGO-GC 的 hook 技术形态确实和外挂注入类似，但行为目标完全不同。CSGO-GC：

- 不读写玩家实体；
- 不修改瞄准、后坐力、视野或输入；
- 不拦截游戏网络指令来获得竞技优势；
- 不隐藏模块；
- 不绕过 auth ticket；
- 不伪造 VAC 状态；
- 主要替换已经失效的 GC 消息接口。

服务器端归属库存、假皮肤或本地 SOCache，影响的是玩家在该服务器/客户端看到和使用的表现，不会把物品写入 Steam 经济系统，也无法：

- 在 Steam 市场出售；
- 发起真实交易；
- 改变 CS2 后台库存；
- 影响 Valve 的物品所有权数据库。

GC 不能让你在对局中获得任何游戏上的优势，也不会让你从官方经济系统凭空获利，Hook 的只是传向 CS:GO 的 GC 消息接口，不会改变游戏本身行为。

## 已知事实

从 2023 年 Mikko 创建此项目开始，公开使用记录中没有已知的、由 CSGO-GC 导致的 VAC 封禁案例。对最终版旧客户端的逆向检查也没有在 `engine.dll` 中找到所搜索的 VAC/secure-mode 字符串。

从 CS:GO 的[独立商店版本](https://store.steampowered.com/app/4465480/CounterStrikeGlobal_Offensive/)来看，CS:GO：
- 不再以 CS2 的 730 身份运行；
- 没有官方服务器、GC、市场等后台系统；
- 公开商店元数据没有声明 VAC enabled；

且 Valve 缺乏继续为这个孤立、停止支持的游戏维护 VAC 策略的现实理由：继续维护 CS:GO 的 VAC 不能为 Valve 带来收益，反而会分担精力和增加成本。

## 未知事项

这些观察不能证明 Valve 不存在未公开的检测或处置路径，也不能预测 VAC 安全服务器或第三方反作弊的行为。“目前没有已知 VAC 封禁”不等于“外挂安全”，也不等于“所有社区服务器都安全”，因为有一些社区服务器使用诸如 GameGuard 的第三方反作弊程序，这类程序可能会扫描内存或者硬盘上的文件，使用带 GC 的客户端游玩这类社区服务器，可能会因 GC 的注入导致第三方封禁。

理论上 Valve 永远保留改变规则或进行人工处罚的能力，但目前没有技术证据表明使用该项目会导致 VAC。这更接近一般性的法律/平台风险保留，而不是发现了某条现实中可能触发 VAC 的代码路径。

## App ID 与 2026 年的独立发行

默认的 `appid_override` 是 `4465480`，即独立的旧版 CS:GO 应用。改成 `730` 会改变 Steam 对运行中游戏的识别方式，并可能影响创意工坊或在线状态；它不会恢复 Valve 的 CS:GO 后端，也不是安全建议。除非你明确了解兼容性代价，否则请保持默认值。

## 我能将 CSGO-GC 用于 CS2 吗？

**强烈建议您不要这样做**。

CSGO 和 CS2 的 GC 格式是相同的（这也就是为什么未使用 CSGO-GC 的游戏客户端，能够通过修改版本号接入 CS2 的 GC），所以理论上，只需要调整启动程序和部分 Hook 端点，这套自定义 GC 也可以用于 CS2。但是，CS2 仍受 Valve 官方支持，这样做**非常有可能**被官方封禁。此行为是**非常危险**的。

## 所以，使用 CSGO-GC 会导致 VAC 封禁吗？

如果你使用带有 CSGO-GC 的客户端自己游玩、和朋友（无论对方是否使用 GC）游玩、游玩同样使用 CSGO-GC 的社区服务器，从技术原理上讲不会导致 VAC 封禁。

如果游玩不带第三方反作弊程序的非 CSGO-GC 社区服务器（即使服务器启用了 VAC），大概率不会被 VAC 封禁。

如果游玩带有第三方反作弊程序的社区服务器，可能会导致第三方的封禁，而第三方封禁是否会连带 VAC 封禁，取决于他们的服务规则。因此，我们不建议您在此类服务器上游玩。
