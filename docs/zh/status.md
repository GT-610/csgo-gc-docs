# 项目状态

本页概括 `.vscode/references/csgo-gc` 中目标项目当前体现出的状态。

## 当前重点

csgo_gc 目前更像是一个面向旧版 CS:GO 的实用本地 GC 替代实现，而不是 Valve 后端的完整克隆。当前实现最成熟的部分集中在库存、物品自定义、本地控制、大厅和专用服务器互操作。

## 已实现

- 通过 `csgo_gc/inventory.txt` 加载和持久化库存。
- 通过 Source RCON 兼容命令实时创建和移除物品。
- 负载和已装备物品更新。
- 武器箱、胶囊、纪念包、涂鸦、命名牌、贴纸、仓库单元、StatTrak 交换和汰换合同等物品使用流程。
- 使用 `csgo_gc/price_sheet.txt` 提供游戏内商店数据和购买完成响应。
- 可配置的账号展示数据，例如段位、称赞、玩家等级、VAC 状态标记、稀有度权重和日志输出。
- ClientGC 和 ServerGC 路径，包括服务器 welcome、客户端 SO cache 转发、服务器侧 SO cache 验证，以及音乐盒 MVP 状态传播。
- 大厅和服务器浏览器支持，并可选择只显示带有 csgo_gc 标记的服务器。
- Windows、Linux 和 macOS 构建目标。

## 已知限制

- 上游项目仍明确标记为未完成。
- 库存边界情况的验证仍需要继续打磨。
- 围绕实时库存编辑的工具仍在计划中；当前核心项目主要通过 RCON 暴露脚本和外部工具接口。
- 匹配系统不在计划内。
- 项目仍希望获得较新的完整 CS:GO Item Schema。

## 安全预期

请把 csgo_gc 当作实验性软件。测试时优先使用单独的 CS:GO 安装目录，或确保可以还原备份。除非你已经添加自己的网络保护，否则请让 RCON 绑定在 localhost。
